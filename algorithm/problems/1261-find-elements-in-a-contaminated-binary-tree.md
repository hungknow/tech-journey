# 1261 Find Elements in a Contaminated Binary Tree

## Problem Description

Given a binary tree with the following properties:
- The `root` of the binary tree is at index `1` in the level order traversal array
- If `root.val == -1`, then `root.left` and `root.right` must also be `-1`
- If `root.val == -1`, then `root.left` must be `-1` and `root.right` can be any value
- If `root.val != -1`, then `root.left` and `root.right` must be non-empty

The binary tree is contaminated and the values in the nodes have been changed to `-1`.

You need to first recover the tree and then implement the `FindElements` class:
- `FindElements(TreeNode* root)` Initializes the internal object with the `root` of the contaminated binary tree.
- `boolean find(int target)` Returns `true` if the `target` value exists in the recovered tree.

```
Output
[null,false,true]
```

## The Twist

The tree is **contaminated with -1 values**. We need to first recover the original tree, then implement efficient search for target values.

## Algorithm

### Tree Recovery:
1. The original values can be recovered from the level order traversal
2. Since all non -1 values become -1, we need to track the original values
3. Use a hash map to map from level order index to original value
4. Reconstruct the tree with original values

### Search Implementation:
1. Store all node values in a hash set for O(1) lookup
2. For each find(target) query, return `true` if target is in the set
3. For efficiency, we could also use a hash map to store values with their indices for range queries

## Complexity

- **Constructor**: O(n) - storing all values
- **find()**: O(1) - hash set lookup
- **Space**: O(n) - storing all values
