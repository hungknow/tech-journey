# 0015 3Sum

## Problem Description

Given an integer array `nums`, return all the triplets `[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, and `j != k`, and `nums[i] + nums[j] + nums[k] == 0`.

Notice that the solution set must not contain duplicate triplets.

### Example 1:
```
Input: nums = [-1,0,1,2,-1,-4]
Output: [[-1,-1,2],[-1,0,1]]
Explanation: 
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0.
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0.
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0.
The distinct triplets are [-1,0,1] and [-1,-1,2].
```

### Example 2:
```
Input: nums = [0,1,1]
Output: []
Explanation: The only possible triplet does not sum up to 0.
```

### Example 3:
```
Input: nums = [0,0,0]
Output: [[0,0,0]]
Explanation: The only possible triplet sums up to 0.
```

## Two Pointers Approach

The key insight is to first sort the array, which allows us to use the two-pointer technique efficiently. For each element `nums[i]`, we can find pairs that sum to `-nums[i]` using two pointers.

### Algorithm Steps:

1. Sort the input array `nums`
2. Iterate through the array with index `i` from 0 to n-3
   - Skip duplicate values of `nums[i]` to avoid duplicate triplets
   - If `nums[i] > 0`, break early (since array is sorted, no further triplet can sum to 0)
   - Use two pointers `left = i+1` and `right = n-1` to find pairs that sum to `-nums[i]`
   - If sum is less than 0, increment `left` pointer
   - If sum is greater than 0, decrement `right` pointer
   - If sum equals 0, add the triplet and skip duplicates for both `left` and `right`

## Complexity

- **Time**: O(n²) - sorting takes O(n log n) and the two-pointer approach takes O(n²)
- **Space**: O(1) - excluding the space for the output

## Solution Code

```go
package main

import (
	"sort"
)

func threeSum(nums []int) [][]int {
	sort.Ints(nums)
	result := make([][]int, 0)
	n := len(nums)
	
	for i := 0; i < n-2; i++ {
		// Skip duplicate values for the first element
		if i > 0 && nums[i] == nums[i-1] {
			continue
		}
		
		// If the current number is positive, break early
		if nums[i] > 0 {
			break
		}
		
		left, right := i+1, n-1
		target := -nums[i]
		
		for left < right {
			sum := nums[left] + nums[right]
			if sum == target {
				result = append(result, []int{nums[i], nums[left], nums[right]})
				
				// Skip duplicates for the second element
				for left < right && nums[left] == nums[left+1] {
					left++
				}
				
				// Skip duplicates for the third element
				for left < right && nums[right] == nums[right-1] {
					right--
				}
				
				left++
				right--
			} else if sum < target {
				left++
			} else {
				right--
			}
		}
	}
	
	return result
}
```
