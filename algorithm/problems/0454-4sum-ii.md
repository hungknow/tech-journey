# 0454 4Sum II

## Problem Description

Given four integer arrays `nums1`, `nums2`, `nums3`, and `nums4`, all of length `n`, return the number of tuples `(i, j, k, l)` such that:
- `0 <= i, j, k, l < n`
- `nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0`

### Example 1:
```
Input: nums1 = [1,2], nums2 = [-2,-1], nums3 = [-1,2], nums4 = [0,2]
Output: 2
Explanation:
The two tuples are:
1. (0, 0, 0, 1) -> nums1[0] + nums2[0] + nums3[0] + nums4[1] = 1 + (-2) + (-1) + 2 = 0
2. (1, 1, 0, 0) -> nums1[1] + nums2[1] + nums3[0] + nums4[0] = 2 + (-1) + (-1) + 0 = 0
```

### Example 2:
```
Input: nums1 = [0], nums2 = [0], nums3 = [0], nums4 = [0]
Output: 1
```

## The Twist

You are summing from **four different arrays**. A brute force O(n⁴) solution is too slow. The key insight is to split the problem into two halves: compute all possible sums from the first two arrays, then find matching complements from the last two arrays.

## Hash Table Usage

- **Key**: `sum(A[i] + B[j])` (sum of elements from nums1 and nums2)
- **Value**: `frequency` (how many pairs produce this sum)

1. Compute all `n²` sums from `nums1` and `nums2`, store in map
2. Iterate through all `n²` pairs from `nums3` and `nums4`
3. For each pair, look up `-(C[k] + D[l])` in the map
4. Sum up all matching frequencies

## Complexity

- **Time**: O(n²) - compute n² sums from first two arrays, then n² lookups
- **Space**: O(n²) - storing up to n² unique sums
